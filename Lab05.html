Lab 5 – AI Integration Reflection

Technical Understanding

What features did I implement with AI assistance??

I thought of four features and asked Claude Sonnet 4.5 to code them for my portfolio. Below are the four features I implemented with Ai assistance:

  • Hamburger Toggle: Allows mobile users to toggle through different navigation tabs.
  • Scroll Reveal Animation: Animates elements to appear/disappear smoothly as users scroll.
  • Light and Dark Mode Toggle: Lets users switch between light and dark modes.
  • Progress Bar: Shows how far the user has scrolled on the page.

How does it work?

I understand the basic mechanics of the code: I tell the AI what I want, and it writes the code. I’m confident with simple functions and if/else logic, but some coding language like threshold, root margin, and class list are still tricky for me to understand.

What did you modify? Did you need to fix bugs, adjust styling, or change functionality?

Hamburger Toggle: I first began with the Hamburger Toggle. This worked perfectly; no changes needed after the prompt was given.
Scroll Reveal Animation: My second feature was my Scroll Reveal feature. Worked as intended and didn't need to add onto the prompt to get my desired outcome.
Light and Dark Mode: Initially too big and unflattering. I modified it to be smaller, removed the sun/moon symbols that it added in the beginning, and added it to all my tabs.
Progress Bar: Took several attempts to make the prompts work; I asked Claude Sonnet 4.5 to change the prompt about 5 times before it gave me a progress bar that worked. Once adjusted, it began to function correctly.

Process Evaluation

When was AI helpful?

AI accelerated the implementation of my different features, I don't know what I would've done without it. My portfolio would look very different if I had to write all the code myself. Ai provided me with a starting point and from there I was able to tweak the code to fit my needs.

When was AI problematic?

For harder features (dark mode toggle, progress bar), AI initially game me code that didn't work or was not fully functional. I had to tweak prompts and make adjustments to get my desired outcome.

What's your role? In an AI-assisted workflow, what remains uniquely human? What can't (or shouldn't) be automated?

My role remains uniquely human: writing prompts, guiding AI, making design decisions, and interpreting how features should behave for users. I believe the human experience and how humans interact with something can never be understood by Ai.

Ethical Considerations

Learning trade-offs: Does vibe coding help or hinder your development as a programmer?

Vibe coding helps me work efficiently with AI, but if I wanted to focus on "fundamentals first", it might hinder my learning and understanding of code.

Attribution: When using AI-generated code, what does proper citation look like?

Proper citation should look the same in academic and professional contexts, I believe in giving the person credit even when it’s extracted by an Ai tool. I think you should always note which parts of the code were Ai assisted and include the tool you used.

Personal guidelines for AI use

I will use AI for debugging, understanding complex code, and generating ideas. I won’t use it for security-sensitive tasks, bypassing rules, or plagiarism.

Prompts Used for Each Feature

  • Hamburger Toggle: Add a hamburger menu to my HTML portfolio. The nav should collapse into a hamburger icon on screens under 768px. Use vanilla JavaScript, no frameworks.
  • Light and Dark Mode:I am building a personal portfolio website using HTML, CSS, and JavaScript, and I want to add a fully functional dark mode toggle. Please implement a clean, modern toggle button in the top-right corner of the navbar labeled “Dark Mode.” The dark mode should be controlled by applying a CSS class such as .dark-mode to the body element, and you should define both light and dark themes using CSS custom properties in root, so the color system is organized and scalable. Make sure the background, text, navbar, buttons, cards or project sections, and links (including hover states) all adjust appropriately when dark mode is enabled. Include smooth transitions between themes for a polished feel.
  • Scroll Reveal Animation: I want to add smooth, professional scroll-based reveal animations to my portfolio website using HTML, CSS, and JavaScript. When elements enter the viewport while scrolling, they should fade in and slightly move upward (subtle translateY effect). The animation should feel clean and modern, not dramatic or distracting. Include smooth easing and clean timing (around 0.6–1s duration). Make sure the animation only triggers once per element (no repeating on scroll up/down unless easily configurable). The code should be production-ready, minimal, and well-commented.
  • Progress Bar: I want to add a thin scroll progress bar to the top of my portfolio website that visually indicates how far the user has scrolled down the page. The progress bar should be fixed to the top of the viewport, span the full width of the screen, and gradually fill from left to right as the user scrolls. The design should be minimal and modern. The bar should be thin (around 3–5px), smooth, and animated with a subtle transition effect. It should not interfere with my existing layout or navigation. Use vanilla JavaScript (no libraries). The progress width should dynamically calculate based on the total scrollable height of the page and update smoothly on scroll. Make sure it works across different screen sizes and handles responsive layouts correctly.

Vibe Coding Guidelines

These are my personal rules for using AI in coding:

  • Use AI for debugging code: I will use Ai to help me identify and fix errors in my code.
  • Use AI to help understand complex code: I will use Ai to help me understand code that may be too difficult for me to read or analyze.
  • Use AI to generate ideas or suggestions: I will use Ai to help me come up with new ideas or suggestions for my projects.
  • Avoid using AI for anything security-related: I will not use Ai for tasks that could compromise security or privacy.
  • Do not use AI to bypass rules or restrictions: I will not use Ai to bypass any paywalls, passwords, or security protections.
  • Do not use AI to plagiarize code or content: I will not use Ai to copy someone else's code or content without proper attribution or their permission.
